# 行业风险分析

## 触发场景

当客户预审、客户准入、尽调、授信判断需要分析目标企业所处行业、主营产品赛道、行业生命周期、政策趋势、市场景气度或行业风险时，执行本 Skill。

## 输入

调用本 Skill 时，优先使用上游上下文中的以下信息：

- 目标企业名称
- 工商经营范围、主营业务、主营产品
- 企业资质、知识产权、股东背景、融资轮次等已查询信息
- 用户明确关注的行业、产品或风险点

若缺少行业或产品关键词，先从经营范围中提炼 1-3 个最能代表主营业务的关键词；仍无法判断时，向用户追问一个最关键问题。

## 工作流

1. 提炼行业检索关键词：
   - 优先使用主营产品、核心技术、经营范围中的具体产品词。
   - 避免只使用"制造业""批发业""技术服务"等过宽行业词。
2. 使用 `searchPersonalKnowledge` 检索行业资料。建议至少检索以下问题：
   - `[行业/产品] 行业发展趋势 政策 监管`
   - `[行业/产品] 市场供需 景气度 竞争格局`
   - `[行业/产品] 信贷风险 周期风险 替代风险`
3. 基于检索结果判断：
   - 行业生命周期：未来发展 / 高增长前景 / 成熟 / 底端通用 / 不确定
   - 赛道价值：高壁垒核心赛道 / 成熟稳定赛道 / 低门槛通用赛道 / 衰退或高风险赛道
   - 主要驱动因素：政策、技术、需求、供给、成本、出口等
   - 主要风险：产能过剩、价格战、政策退坡、技术替代、下游需求波动、合规监管等
4. 若知识检索结果不足，必须说明资料不足，不得编造行业数据；可以结合企业经营范围和一般产业逻辑给出保守判断。

## 输出要求

输出一段可被客户预审报告直接使用的结构化结果：

```text
行业关键词：
生命周期判断：
赛道价值判断：
主要驱动因素：
主要风险：
行业定性结论（200字以内）：
资料充分性：充分 / 部分充分 / 不足
```

行业定性结论应服务于客户预审评级 Step 2，明确说明该行业对企业信用风险的正向或负向影响。
